DeepIM: Deep Iterative Matching for 6D Pose Estimation

Yi Li · Gu Wang · Xiangyang Ji · Yu Xiang · Dieter Fox

https://arxiv.org/pdf/1804.00175.pdf

Introduction

 

The DeepIM model is a deep neural network that uses a combination of convolutional and recurrent neural networks to estimate the 6DoF pose of an object given an input image and a 3D model of the object. The model works by iteratively refining the estimated pose using a process called iterative matching.

 

In the first stage, the model uses a convolutional neural network (CNN) to extract features from the input image and the 3D model. These features are then used to generate a set of 2D keypoint correspondences between the image and the model.

In the second stage, the model uses a recurrent neural network (RNN) to estimate the 6DoF pose of the object from the set of keypoint correspondences. The RNN iteratively refines the pose estimate by predicting a new set of keypoint correspondences and using them to update the pose estimate.

 

The DeepIM model has been shown to achieve state-of-the-art performance on several benchmark datasets for 6DoF pose estimation, including the LINEMOD and Occlusion datasets.

 

Required Inputs

 

The required inputs for the DeepIM model for 6DoF pose estimation include the following:

  1. A 3D model of the object: This is a computer-aided design (CAD) model of the object that needs to be detected and tracked. The model is typically represented as a set of vertices and faces, along with the object's physical dimensions.
  2. An input image or a set of images: These are the images that contain the object in the scene. The images can be captured from different viewpoints, and the model can be rendered into the images using a virtual camera with known extrinsic parameters.
  3. Camera intrinsics and extrinsics: These are the parameters that describe the geometry and orientation of the camera(s) used to capture the input images. The camera intrinsics include the focal length, principal point, and distortion coefficients, while the extrinsics include the camera pose and orientation in the 3D world coordinate system.
  4. Ground truth poses: These are the known ground truth poses of the object in the scene, which are used to train the model. The ground truth poses can be obtained by manual annotation or by using a motion capture system.

 

It is important to note that the DeepIM model is trained using pairs of input images and 3D models, along with their corresponding ground truth poses. During inference, the model takes an input image and the 3D model of the object as input and outputs the estimated 6DoF pose of the object.

 

Outputs

 

The output of the DeepIM model for 6DoF pose estimation is the estimated 6DoF pose of the object in the input image(s) with respect to the camera coordinate system. The pose is represented as a 4x4 transformation matrix that specifies the translation and rotation of the object in 3D space.

 

The pose matrix has the following form:

 

[[R11, R12, R13, t1],

 [R21, R22, R23, t2],

 [R31, R32, R33, t3],

 [0,   0,   0,   1 ]]

 

where R is a 3x3 rotation matrix and t is a 3x1 translation vector. The rotation matrix specifies the orientation of the object in 3D space, while the translation vector specifies the position of the object in the camera coordinate system.

 

The estimated pose can be used for a variety of applications, such as object tracking, augmented reality, and robotics.
